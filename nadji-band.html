Website find band

Web application used for connecting musicians, bands and taverns with shared interests. Technologies and frameworks used: React, Node.js, Mongoose, Express, MongoDB. Source code can be found:
I have published it using heroku, and can be previewed on:

Home page with a Navbar on top, slider with buttons in body and Footer on bottom of the page. When user clicks Register button, it shows up registration form.

On this form, user must choose account type ( musician, band or tavern), enter his/her email adress, and password two times. On front end side, there are checks did user choose account type, did user enter a valid email adress and checks for password equality. If everything is correct, then that information is passed to the server. On server side, first, it checks is that email already taken, and if it is, server response with a message "Email already exists" If it is not taken, on server side is made the secret token, which is sent on provided email and as response server redirects user to a page with a form where user should enter a secret token which he/she recieved on email. Server side also save that user in database, with extra field active, which is setted on false by default. Password is crypted with bcrypt library. If user enters a valid secret token, on server side active field in databse of that account is changed from false to true and as response redirects user to configure your profile page.

Configure your profile page is represented as form, where user can add some more information which depends on account type. For example, musician can add information about genre, instrument, location, description and purpose of making account. At the end of form, there are two buttons, save changes and delete account. Save changes button saves everything in database and redirects user to view profile page. Delete button deletes that user, and everything what did that user made (all ads) and redirects user to home page.

On this page, user can see how does it look his/her profile page. A new things are profile image, and stars. Stars represents a ratings from other users. Now, when user configure profile, from navbar he/she can list all Musicians, Bands, or Taverns. Separately for each page for listing ads, there are some filters for easier search on left side, and list of ads on right side. Every ad has title, description, location, email adress and some other fields which depends who made ad (Musician, Band or Tavern). Also, there are two buttons:

  1. Contact
  2. View profile

If user clicks button View profile, he/she is redirected to profile of user who posted that ad. User can see profile, and also rate that profile. If user clicks Contact button, he/she is redirected to page with contact form. User should enter a title of message, and message which will be sent to the user who posted ad. At navbar, user can choose CreateAd, which redirects to the form from where user can make her/his own ad.

At the end, I would like to say something about implementation of server side. All data were saved on MongoDB. There are 3 collections in database:

  1. Users
  2. Forums
  3. Ratings
Each of them has their own controller for GET and POST methods (saving, updating, getting and removing data from database), which works with the help of frameworks:
  1. Mongoose
  2. Express
